# 웨딩 청첩장 + 청첩장 배달 서비스 개발 기획 (최종 — 참여 시스템)

- **GitHub:** https://github.com/blackshoes1/invitation-one
- **스택:** Next.js (App Router), Tailwind CSS v4, TypeScript, Supabase
- **모바일 청첩장 URL:** invitation-one-three.vercel.app (`/`)

> **이번 개정의 핵심: 그룹 시스템 → 참여 시스템**
> 그룹(단톡방 링크)은 유지하되, 하객 행동의 중심축이 "주문 생성"에서 **"참여(합류)"** 로 이동.
> 마음 배송도 별도 테이블이 아니라 **참여의 한 형태**(`participants.type`)로 통합.

---

## 0. 전체 컨셉 & 플로우 (가장 중요)

**핵심:** 배달 링크가 시작점, 모바일 청첩장은 "만남 이후의 보상"

### 메인 플로우 — 친구·동료 그룹

```
배달 링크 먼저 공유 (모바일 청첩장 링크 노출 안 함)
       ↓
하객이 두 갈래 중 선택
  ├─ 직접 배달 받기 🛵 → 날짜/장소 신청 or 기존 주문 합류 → 모임에서 종이 청첩장(QR) 전달
  │                     → QR 스캔 시 모바일 청첩장 공개 🎉
  └─ 마음 배송 💌      → 축하 메시지 + 지역 남기기 → 바로 모바일 청첩장 공개
```

### 예외 플로우 — 어르신·친척 그룹

모바일 청첩장 링크(접근 키 포함)를 바로 전달 (심플하게)

**설계 원칙**

- 모바일 청첩장은 비공개가 아니라 *공개 시점만* 모임 이후로 설계
- "마음 배송"(못 만나는 분)은 그 자리에서 바로 모바일 청첩장 확인 가능 → 서운함 없음

## 1. 모바일 청첩장 (기존 사이트 개편)

### 페이지 순서 (10개 → 7개 블록으로 정리)

```
Hero → Greeting → Gallery → D-Day → Location
→ 💝 축하해준 사람들 (지도/메시지 통합)
→ 마음 전하기(계좌) → 참석 의사 전하기
```

구조: 청첩장 본체(Hero~Location) → 축하의 흔적(통합) → 하객의 행동(계좌·참석의사)

### 💝 축하해준 사람들 (배송 여정 지도 + 방명록 + 리뷰 통합)

```
💝 우리를 축하해준 사람들
"지금까지 32명이 함께해주셨어요"

[지도 🗺️] / [메시지 💬]  ← 두 뷰 전환
```

**지도 뷰:** 두 종류의 핀이 실시간으로 쌓임 ← *(개정)*

- **🛵 핀 — 직접 배달 완료** (좌표: `delivery.location` 기반, 배송 완료 시점에 생성)
  - 핀 탭: "8/15 홍길동님을 만났어요"
- **💌 핀 — 마음 배송** (좌표: `region` 매핑, 신청 즉시 생성)
  - 핀 탭: "부산에서 김영희님의 마음이 도착했어요"
- **범례 + 필터:** 지도 상단 "🛵 직접 만난 분 12 · 💌 마음 보낸 분 20" — 탭하면 해당 핀만 보기.
  메시지 뷰의 뱃지 체계(🛵✅/💌)와 아이콘 일치 → 학습 비용 없음
- **같은 지역 겹침 처리:** 💌는 시/군/구 좌표라 같은 지역 여러 명 겹침 → 핀 하나에 "💌 3" 배지 → 탭하면 명단
- **본인 핀 강조:** 본인 확인된 사람의 핀만 크게 + 강조 ("당신은 N번째 손님이었어요")
- 프라이버시: 이름+날짜+지역(시/군/구 수준)만, 정확한 주소 전체 노출 안 함

**메시지 뷰:** 방명록(마음 배송) + 리뷰(직접 배달)를 한 줄기로 섞어 시간순 정렬

```
🛵✅ 홍길동  ⭐⭐⭐⭐⭐  "신랑 친절하고 청첩장 예뻐요"
💌 김영희 (부산 해운대구)  "멀리 있어 못 가지만 축하해 🎉"
🛵✅ 박철수  "직접 받으니 감동이었어요"
```

- 뱃지(🛵✅/💌)·별점 유무로 자연스럽게 구분, 연락처 비노출
- **데이터: `participants` 단일 테이블에서 조회** — 마음배송 행(스탬프/메시지) + 직접배달 완료 행(리뷰)을 `created_at` 기준 정렬 ← *(개정: messages 테이블 머지 불필요)*
- 초반(핀·리뷰 적을 때) 대비 문구 필요 (오픈 직후엔 데이터 0 → 별도 안내)

### 마음 전하기 (계좌이체 개선)

- 기본은 접혀있고 "마음 전하기"를 눌러야 펼쳐짐 (부담 주지 않는 톤, "축의대 미운영" 문구와 결 맞춤)
- 신랑측 / 신부측 각각 버튼 2개로 압축:
  - **[계좌번호 복사]** — "OO은행 1234567890 성근영" 통째로 복사 + "복사됐어요!" 토스트 (기본·필수)
  - **[카카오페이로 보내기]** — 카카오페이 코드송금 링크(qr.kakaopay.com/...). 비즈니스 계정 불필요. 금액 미지정으로 발급. 어디까지나 보조
- 은행 앱 자동 실행 방식은 **제외** (앱 스킴 불안정)
- 핵심은 "복사 한 번 → 본인 은행 앱에서 붙여넣기"

### 참석 의사 전하기 (기존 RSVP 개편)

명칭: "RSVP" → "참석 의사 전하기"

```
성함 [________]
구분  [신랑측] [신부측]

참석하시나요?
[참석합니다] [마음으로 함께해요]

(참석 선택 시 펼쳐짐)
동반 인원 (본인 제외) [0][1][2][3][4+]
└ 자녀 동반 시 인원 + 유아식 필요 여부
식사 여부 [식사합니다][식사 안 해요][미정]
```

- 핵심: 혼주가 실제 쓰는 식수 데이터 정확히 집계 (성인/자녀 구분), 신랑측/신부측 구분 통계
- 중복 방지: 연락처+이름 기준 1회. 재제출 시 기존 내용 수정
- 배달 신청과 역할 분리: 배달 = 청첩장 받기(식 전 만남) / 참석 의사 = 당일 참석·식사(운영용)

### 접근 제어

- QR 코드에 접근 키 파라미터 포함 (예: `/?key=xxxx&via=qr`)
- 키 없이 메인 접근 시 → "아직 공개 전이에요 😊" 안내 + 배달 페이지로 안내
- 어르신 그룹에는 키 포함 링크를 바로 전달 (`via` 없음 → 본인 확인 UI 미노출)
- ※ 완벽한 보안 아님. 청첩장 특성상 이 정도면 충분, 로그인 불필요

### 본인 확인 & 뱃지 (QR로 진입한 하객 개인화)

- 종이 청첩장 QR은 완전 공통 (개인별/날짜별 아님 → 인쇄 한 번에 끝)
- 청첩장 열람 후 본인 확인:

```
성함 [____] + 전화번호 끝 4자리 [____]
→ participants에서 (이름 + 전화 끝4)로 조회  ← (개정: deliveries가 아니라 participants)
   · 1명 매칭: 바로 인증 완료
   · 2명 이상 매칭: 만난 날짜(청모 날짜) 추가 입력받아 구분 (충돌 시에만)
→ 조회 결과로 청모 날짜·리뷰·뱃지 타입 자동으로 딸려옴
```

- 뱃지 종류
  - 직접 배달받은 하객(type=직접배달) → "직접 만나 받은 특별한 분 🛵✅"
  - 마음 배송한 하객(type=마음배송) → "마음으로 함께한 분 💌"
- **마음 배송은 연락처가 선택 입력** ← *(개정)*: 남기면 재방문 시 본인 확인·뱃지 가능, 안 남기면 그 세션에서만
- QR 진입자(`via=qr`)에게만 본인 확인 UI 표시. 키 직접 전달 그룹은 미노출

### Gallery 섹션 (신규)

- 커플 사진 2~3장 슬라이드, `/public/pic/` 폴더

### 팝업/플로팅 버튼 → 제거

- 모바일 청첩장에서 배달로 유도하지 않음 (플로우 역전)

### 디자인 톤 통일 가이드

- 청첩장(세이지/베이지) ↔ 배달(밝고 경쾌) 분위기는 다르게, 로고/이름 폰트(Noto Serif)와 메인 포인트 컬러 1개는 공통

## 2. 청첩장 배달 서비스 (신규) — 시작점

### URL 구조

```
/delivery                         ← 메인 (그룹 없이 접근 시 — 새 주문 전용)
/delivery/group/[groupId]         ← 그룹별 비공개 페이지 (합류가 기본 동선)
/delivery/manage/[participantId]  ← 신청 취소/변경 (참여자 단위)  ← (개정: deliveryId → participantId 통일)
```

- **비그룹 진입(`/delivery`)은 새 주문 전용** ← *(개정)*: 프라이버시상 전체 주문 목록을 노출할 수 없으므로 합류 동선은 그룹 페이지에서만

## 3. 배달 메인 — 두 갈래 메뉴

### 첫 진입 연출 (첫 3초)

- 인트로 애니메이션 1회 (🛵 + "배송 출발!" 도장), 1.5~2초, 스킵 가능, sessionStorage 1회

```
🛵 메뉴를 골라주세요

[직접 배달 받기 🛵]   — 만나서 청첩장 받기
[마음 배송 💌]        — 못 만나도 마음 전하기
```

### 리뷰 노출로 신청 유도

- 메뉴/신청 직전 화면에 공개 리뷰가 흐름 — "먼저 받은 분들이 이렇게 말했어요 ⭐"
- 초반 대비: 리뷰 0개면 영역 숨김 ("첫 손님을 기다리고 있어요 🛵")
- 배달 신청이 메인, 마음 배송은 보조. "거절하기" 같은 직접 표현 금지

### 폼 헤더 (공통)

```
🛵 성근영·김아영 스토어
[그룹명 — 그룹 페이지인 경우]
⭐ 신규 오픈 · 무료배송
📦 남은 자리: 00개 (전체 통합 기준)
```

## 4. 직접 배달 받기 — 두 가지 흐름

### A. 새 주문 (첫 신청 또는 다른 날짜 제안) — 스텝 폼

```
1단계: 받는 분 정보 (이름 + 연락처 한 화면) 📋📞
2단계: 배송지를 입력해주세요 📍 (전국 어디든 🛵)
3단계: 배송 희망일 📅 — 7/6~10/16, 예약일 비활성화 (전체 통합)
4단계: 배송 희망 시간대 ⏰ [🌅 오전] [☀️ 오후] [🌙 저녁]
5단계: 배송기사(신랑) 요청사항 💬 (선택)
       ↓
[신청 직전 요약 확인]
받는 분 / 배송지 / 배송 예정 / 함께 받는 인원(자동)
[수정하기] [네, 주문할게요 🛵]
```

- ※ 예상 인원 입력 단계 **삭제** — 참여자 수로 자동 집계

### B. 합류 (기존 주문에 함께 받기) — 초경량 폼

```
8/5(수) 저녁 주문에 합류할게요 ➕
성함 📋 [________] / 연락처 📞 [________]
[함께 받기 🛵]
  ↓ 합류 완료! 홍길동님 외 3명과 함께 받아요 🎉
```

- 날짜/장소/시간은 기존 주문 것을 따름 → 입력 최소화
- 합류 후에도 갈아타기/나가기 가능 (manage 화면)

## 5. 마음 배송 — 메시지 폼

```
💌 마음 배송

축하 스탬프 (탭 한 번)  🎉 💐 🥂 ❤️ 🙏
이름 [________]
어디서 마음을 보내시나요? 📍 [시/도 ▼] [시/군/구 ▼]   ← (개정: 필수)
한마디 (선택) [________________]
연락처 (선택) [________]  ← 재방문 때 알아볼 수 있게 (뱃지·본인확인용)

[마음 전하기 💌]
       ↓
따뜻한 마음 잘 받았어요 🥰 → 감사 영상 → [모바일 청첩장 보기] 바로 공개
```

- **지역 입력 방식** ← *(개정)*: 자유 텍스트가 아니라 **시/도 → 시/군/구 2단계 선택** + "해외 🌍" 옵션(국가명 입력)
  - 프라이버시 규칙(시/군/구 수준)이 입력 단계에서 자동 보장
  - 지도 핀 좌표를 지역명→좌표 고정 매핑으로 처리 (지오코딩 불필요)
  - 의미 없는 입력 원천 차단
- 폼 문구는 "배송지"가 아니라 "어디서 마음을 보내시나요?" 톤 (실제 배달 없음)
- 남긴 메시지는 청첩장 방명록 + 💌 지도 핀으로 표시
- 신랑신부는 Admin에서 확인 후 카톡으로 직접 답장 (자동화 X)

### 마음 ↔ 직접 배달 전환

- `participants.type` 변경으로 처리 ← *(개정: 테이블 간 이관 없음)*

- 직접 배달 → 사정 생기면 → 마음 배송으로 부드럽게 전환 (manage 화면에서)

## 9. 참여 시스템 (구 그룹 시스템) — 합류 + 다른 일정 제안

```
Admin에서 그룹 생성 → 고유 링크 (/delivery/group/abc123) → 단톡방 공유
       ↓
하객이 링크 열면 그룹 페이지:
  · 기존 주문 있음 → [여기 합류하기 ➕] (기본 동선)
  · 날짜가 안 맞음 → [다른 날짜 제안하기] (= 새 주문 생성)
  · 주문 없음 → 첫 주문 생성
```

### 그룹 페이지 — 주문 단위 현황 (배송 상태 실시간 연동)

```
─── 우리 그룹 주문 현황 ───

🛵 8/5(수) 저녁 — 홍길동 외 2명 · 👨‍🍳 준비 중
   참여: 홍길동, 김철수, 박민수
   [여기 합류하기 ➕]

🛵 8/9(일) 오후 — 이영희 외 1명 · 📦 주문 접수
   [여기 합류하기 ➕]

다 안 맞아요 → [다른 날짜 제안하기]
(이름 + 날짜 + 배송 단계만 / 연락처·배송지는 비공개)
```

### 핵심 규칙

- **합류:** 이름+연락처만 입력하면 기존 주문에 참여자로 추가
- **다른 날짜 제안 = 새 주문:** 그룹 안에 주문이 여러 개 공존
- **갈아타기:** 탭 한 번으로 다른 주문으로 이동 → 그룹이 자연스럽게 인원 많은 날짜로 수렴
- **자동 취소:** 참여자 0명 주문은 자동 취소 + 슬롯 즉시 해제
- **여러 주문 허용:** 한 그룹이 2개 이상 모임으로 나뉘어도 OK
- 인원은 참여자 수로 **자동 집계**
- 같은 그룹이 서로의 배송 단계를 보며 단톡방 대화거리 생성

### 그룹 특성

- 비공개(링크 아는 사람만), 인원 제한 없음, 다른 그룹 정보 열람 불가

### ⚠️ 중요 — 날짜 슬롯은 전체 통합 관리 (변동 없음)

- 그룹 무관 전체 날짜 풀(7/6~10/16, 103일)에서 하루 1주문
- 합류는 기존 주문에 붙는 것이므로 슬롯을 새로 차지하지 않음

## 10. 입력값 Validation

- **이름:** 빈 값 방지, 최소 2자
- **연락처:** 010-XXXX-XXXX 형식 + 자동 하이픈 (마음 배송은 선택 — 입력 시에만 검증)
- **장소:** 빈 값/최소 길이 체크
- **지역(마음 배송):** 시/도·시/군/구 미선택 시 진행 불가 ← *(개정)*
- **날짜:** 마감일 비활성화, 범위 밖 노출 안 함
- **시간대:** 미선택 시 진행 불가
- **요청사항/메시지:** 선택, 최대 500자
- **합류:** 같은 주문에 같은 (이름+연락처) 중복 합류 방지 → "이미 이 주문에 함께하고 계세요 😊"
- **동시성:** `deliveries.date` 조건부 unique(status != '취소'). 실패 시 "방금 다른 분이 먼저 신청하셨어요 😢" → 날짜 단계 복귀 + 재조회

## 14. 우선순위 (개발 순서)

1. **participants 테이블 + 마이그레이션 + 전체 통합 슬롯 로직** ← *(개정: 그룹 테이블 → participants 승격)*
2. 배달 메인(메뉴 분기 + 리뷰 노출) + 그룹 페이지(주문 현황 + 합류)
3. 직접 배달 스텝 폼 + 요약 확인 + 완료 화면 / 합류 폼
4. 마음 배송 폼(지역 필수) + 방명록 연동
5. 모바일 청첩장 개편(축하해준 사람들 — 이원 핀 지도, 접근 키, 본인확인·뱃지)
6. 취소/변경 페이지(참여자 단위) + 배송 추적
7. Admin (참여자 뷰, 주문/메시지 관리)
8. SMS(솔라피), 영상 링크, 대기자, .ics 등 부가 기능
