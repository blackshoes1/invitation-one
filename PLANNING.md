# 웨딩 청첩장 + 청첩장 배달 서비스 개발 기획 (최종)

- **GitHub:** https://github.com/blackshoes1/invitation-one
- **스택:** Next.js (App Router), Tailwind CSS v4, TypeScript, Supabase
- **모바일 청첩장 URL:** invitation-one-three.vercel.app (`/`)

## 0. 전체 컨셉 & 플로우 (가장 중요)

**핵심:** 배달 링크가 시작점, 모바일 청첩장은 "만남 이후의 보상"

### 메인 플로우 — 친구·동료 그룹

```
배달 링크 먼저 공유 (모바일 청첩장 링크 노출 안 함)
       ↓
하객이 두 갈래 중 선택
  ├─ 직접 배달 받기 🛵 → 날짜/장소 신청 → 모임에서 종이 청첩장(QR) 전달
  │                     → QR 스캔 시 모바일 청첩장 공개 🎉
  └─ 마음 배송 💌      → 축하 메시지 남기기 → 바로 모바일 청첩장 공개
```

### 예외 플로우 — 어르신·친척 그룹

모바일 청첩장 링크(접근 키 포함)를 바로 전달 (심플하게)

**설계 원칙**

- 모바일 청첩장은 비공개가 아니라 *공개 시점만* 모임 이후로 설계
- "마음 배송"(못 만나는 분)은 그 자리에서 바로 모바일 청첩장 확인 가능 → 서운함 없음

## 1. 모바일 청첩장 (기존 사이트 개편)

### 페이지 순서 (10개 → 7개 블록으로 정리)

```
Hero → Greeting → Gallery → D-Day → Location
→ 💝 축하해준 사람들 (지도/메시지 통합)
→ 마음 전하기(계좌) → 참석 의사 전하기
```

구조: 청첩장 본체(Hero~Location) → 축하의 흔적(통합) → 하객의 행동(계좌·참석의사)

### 💝 축하해준 사람들 (배송 여정 지도 + 방명록 + 리뷰 통합)

```
💝 우리를 축하해준 사람들
"지금까지 32명이 함께해주셨어요"

[지도 🗺️] / [메시지 💬]  ← 두 뷰 전환
```

**지도 뷰:** 배송 완료 핀이 실시간으로 쌓임

- 핀 탭: 날짜 + "OO님을 만났어요"
- 본인 확인된 하객: "당신은 N번째 손님이었어요" + 자기 핀 강조
- 프라이버시: 이름+날짜만, 정확한 주소 전체 노출 안 함 (동/역 수준까지만)

**메시지 뷰:** 방명록(마음 배송) + 리뷰(직접 배달)를 한 줄기로 섞어 시간순 정렬

```
🛵✅ 홍길동  ⭐⭐⭐⭐⭐  "신랑 친절하고 청첩장 예뻐요"
💌 김영희  "멀리 있어 못 가지만 축하해 🎉"
🛵✅ 박철수  "직접 받으니 감동이었어요"
```

- 뱃지(🛵✅/💌)·별점 유무로 자연스럽게 구분, 연락처 비노출
- 데이터: `messages` 테이블 + `deliveries`의 review 필드를 합쳐 `created_at` 기준 정렬
- 초반(핀·리뷰 적을 때) 대비 문구 필요 (오픈 직후엔 데이터 0 → 별도 안내)

### 마음 전하기 (계좌이체 개선)

- 기본은 접혀있고 "마음 전하기"를 눌러야 펼쳐짐 (부담 주지 않는 톤, "축의대 미운영" 문구와 결 맞춤)
- 신랑측 / 신부측 각각 버튼 2개로 압축:
  - **[계좌번호 복사]** — "OO은행 1234567890 성근영" 통째로 복사 + "복사됐어요!" 토스트 (기본·필수)
  - **[카카오페이로 보내기]** — 카카오페이 코드송금 링크(qr.kakaopay.com/...). 비즈니스 계정 불필요, 개인이 카카오페이 앱/개발자센터에서 발급 가능. 금액 미지정으로 발급(하객이 직접 입력). PC·앱 미설치 시 매끄럽지 않으므로 어디까지나 보조
- 은행 선택 → 은행 앱 자동 실행 방식은 **제외** (앱 스킴은 계좌·금액 자동 입력 불가, 은행마다 다르고 수시로 바뀜, 미설치 시 깨짐)
- 핵심은 "복사 한 번 → 본인 은행 앱에서 붙여넣기"가 가장 안 깨지고 확실

### 참석 의사 전하기 (기존 RSVP 개편)

명칭: "RSVP" → "참석 의사 전하기"

```
참석 의사 전하기

성함 [________]
구분  [신랑측] [신부측]

참석하시나요?
[참석합니다] [마음으로 함께해요]

(참석 선택 시 펼쳐짐)
동반 인원 (본인 제외) [0][1][2][3][4+]
└ 자녀 동반 시 인원 + 유아식 필요 여부
식사 여부 [식사합니다][식사 안 해요][미정]
```

- 핵심: 혼주가 실제 쓰는 식수 데이터 정확히 집계 (성인/자녀 구분)
- 신랑측/신부측 구분 통계
- 중복 방지: 연락처+이름 기준 1회. 재제출 시 "이미 참석 의사를 전해주셨어요 😊 수정하시겠어요?" → 기존 내용 수정
- 배달 신청과 역할 분리: 배달 = 청첩장 받기(식 전 만남) / 참석 의사 = 당일 참석·식사(운영용)

### 접근 제어

- QR 코드에 접근 키 파라미터 포함 (예: `/?key=xxxx`)
- 키 없이 메인 접근 시 → "아직 공개 전이에요 😊" 안내 또는 배달 페이지로 안내
- 어르신 그룹에는 키 포함 링크를 바로 전달
- ※ 완벽한 보안 아님(공유되면 키도 노출). 청첩장 특성상 이 정도면 충분, 로그인은 불필요

### 본인 확인 & 뱃지 (QR로 진입한 하객 개인화)

- 종이 청첩장 QR은 완전 공통 (개인별/날짜별 아님 → 인쇄 한 번에 끝)
- 청첩장 열람 후 본인 확인:

```
성함 [____] + 전화번호 끝 4자리 [____]
→ deliveries에서 (이름 + 전화 끝4)로 조회
   · 1명 매칭: 바로 인증 완료
   · 2명 이상 매칭(동명이인+끝4 우연 일치): "만나신 날짜를 확인할게요"
     → 청모 날짜(yymmdd) 추가 입력받아 구분 (평소엔 안 물어봄, 충돌 시에만)
→ 조회 결과로 청모 날짜·주문번호·리뷰 자동으로 딸려옴
```

- 인증되면 개인화: "홍길동님, 8/15에 만나서 반가웠어요!" + 뱃지 표시
- 뱃지 종류
  - 직접 배달받은 하객 → "직접 만나 받은 특별한 분 🛵✅"
  - 마음 배송한 하객 → "마음으로 함께한 분 💌" (신청 완료 시점에 바로 부여)
- 뱃지는 모바일 청첩장에 계속 노출 (QR로 본인 확인하고 들어온 사람 기준)
- 어르신/키 직접 전달 그룹 분기: 배달 없이 접근 키 링크로 바로 들어온 사람은 `deliveries`에 데이터 없음 → 본인 확인란·뱃지·"N번째 손님"을 아예 노출하지 않음 (본인 확인 UI는 QR 진입자에게만 표시)

### Gallery 섹션 (신규)

- 커플 사진 2~3장 슬라이드
- 사진은 `/public/pic/` 폴더에 추가 예정

### 팝업/플로팅 버튼 → 제거

- 더 이상 모바일 청첩장에서 배달로 유도하지 않음 (플로우가 역전됨)

### 디자인 톤 통일 가이드

- 모바일 청첩장(세이지/베이지 감성)과 배달 페이지(밝고 경쾌한 톤)는 분위기를 다르게 가져가되, 로고/이름 폰트(Noto Serif)와 메인 포인트 컬러 1개는 공통 적용 → "같은 청첩장" 신뢰 유지

## 3. 배달 메인 — 두 갈래 메뉴

### 첫 진입 연출 (첫 3초)

- 페이지 열리면 짧은 인트로 애니메이션 1회 (🛵가 가로질러 들어오고 "배송 출발!" 도장이 쾅 → 메인 등장)
- 1.5~2초, 스킵 가능, `sessionStorage`로 1회만 노출
- (진심 글 자리 — 나중에 추가 예정. 메뉴 선택 바로 위에 배치)

```
🛵 메뉴를 골라주세요

[직접 배달 받기 🛵]   — 만나서 청첩장 받기
[마음 배송 💌]        — 못 만나도 마음 전하기
```

### 리뷰 노출로 신청 유도

- 메뉴/신청 직전 화면에 먼저 받은 하객들의 공개 리뷰가 흐름
- "먼저 받은 분들이 이렇게 말했어요 ⭐" → 아직 신청 안 한 사람의 신청 동기 자극 (리뷰 → 신청의 선순환)
- 초반 대비: 오픈 직후엔 리뷰 0개 → 리뷰 N개 미만이면 영역 숨김 또는 신랑신부가 쓴 안내 문구로 대체 ("첫 손님을 기다리고 있어요 🛵")
- 배달 신청이 메인, 마음 배송은 보조 (작게/아래 배치) → 신청률 유지
- "마음 배송"은 거절이 아니라 또 하나의 축하 방식으로 프레이밍. 문구는 "거절하기" 같은 직접 표현 금지

### 폼 헤더 (공통)

```
🛵 성근영·김아영 스토어
[그룹명 — 그룹 페이지인 경우]
⭐ 신규 오픈 · 무료배송
📦 남은 자리: 00개 (전체 통합 기준)
```

## 4. 직접 배달 받기 — 스텝 폼 (진행률 도트 ● ● ○ ○)

```
1단계: 받는 분 정보 (이름 + 연락처 한 화면)
       성함 📋 / 배송 완료 후 연락드릴 번호 📞
2단계: 배송지를 입력해주세요 📍
       (전국 어디든 직접 배달합니다 🛵 / 정확할수록 빨리 찾아가요)
3단계: 배송 희망일을 선택해주세요 📅
       7/6 ~ 10/16, 예약된 날짜 비활성화 (전체 통합 기준)
       (● 마감  ○ 배송 가능 / 가장 빠른 가능일 기본 포커스 / 주말만 보기 필터)
4단계: 배송 희망 시간대 ⏰  [🌅 오전] [☀️ 오후] [🌙 저녁]
5단계: 당일 예상 참석 인원 👥 (그룹 만남 인원 파악용)
6단계: 배송기사(신랑)에게 요청사항 💬 (선택, 예: 신부도 같이 와주세요)
       ↓
[신청 직전 요약 확인 화면]
이대로 주문할까요?
받는 분 / 배송지 / 배송 예정 / 예상 인원
[수정하기] [네, 주문할게요 🛵]
```

## 5. 마음 배송 — 메시지 폼

```
💌 마음 배송

축하 스탬프 (탭 한 번)
🎉 💐 🥂 ❤️ 🙏

이름 [________]
한마디 (선택) [________________]

[마음 전하기 💌]
       ↓
따뜻한 마음 잘 받았어요 🥰

직접 못 뵙는 게 아쉬워서
저희가 짧게 인사 남겼어요
[두 사람의 짧은 감사 영상] → 유튜브 비공개 링크

결혼식에서는 꼭 안아드릴게요!
       ↓
[모바일 청첩장 보기] → 바로 공개
```

- 우편 옵션은 제외 (메시지에 집중)
- 남긴 메시지는 모바일 청첩장 방명록에 표시
- 직접 배달의 영상과는 다른, "못 만나서 미안한 마음"에 답하는 별도 영상
- 신랑신부는 Admin에서 확인 후 카톡으로 직접 답장 (자동화 X)

### 마음 ↔ 직접 배달 전환

- 마음 배송한 사람이 나중에 "역시 만나고 싶어요" → 직접 배달로 전환 가능
- 직접 배달 신청했다가 사정 생기면 → 마음 배송으로 부드럽게 전환
- 거절이 끝이 아니라 언제든 마음이 바뀔 여지를 주는 흐름 (완료/manage 화면에서 전환 링크 제공)

### 배송 현황 추적 (콘셉트 재미 — 실시간)

Admin에서 단계를 수동으로 넘기면 하객 화면에 실시간 반영

```
📦 주문 접수   — 신청 완료 (대기중)
👨‍🍳 준비 중    — 확정, 청첩장 챙기는 중 (확정)
🛵 배송 출발   — 만나러 출발! "🛵 신랑이 출발했어요! 예상 도착 10분 전"
✅ 배송 완료   — 전달 끝 (완료)
```

- 실제 위치 추적 아님, 단계만 수동 전환

### 리뷰 남기기 (배송 완료 후)

`status`가 '완료'로 바뀌면 리뷰 화면 열림

```
청첩장은 만족스러우셨나요? ⭐⭐⭐⭐⭐

한줄 리뷰 (선택)
[________________]
예) 신랑 친절해요 / 청첩장 예뻐요 / 별 다섯 개 드려요

[리뷰 등록하기]
```

- 리뷰는 공개 — 배달 페이지 "고객 리뷰" 섹션 + 모바일 청첩장에 노출
- 결혼식 당일 영상 소스로도 활용 가능
- 방명록(마음 배송 메시지)과는 다른 결의 콘텐츠 (직접 만난 후기)

## 9. 그룹 시스템

```
신랑신부가 Admin에서 그룹 생성 (대학 친구들, 직장 동료, 가족 등)
       ↓
그룹별 고유 링크 자동 생성 (/delivery/group/abc123)
       ↓
각 그룹 카카오톡 단톡방에 링크 공유
       ↓
하객이 링크 열면 해당 그룹 페이지 → 메뉴 선택 → 신청
```

### 그룹 페이지 — 멤버 신청 현황 공개 (배송 상태 실시간 연동)

```
─── 우리 그룹 신청 현황 ───
✅ 홍길동 — 8/5 오후 · 🛵 배송 출발
✅ 김철수 — 8/12 오전 · ✅ 배송 완료
⬜ 이영희 — 미신청
(이름 + 날짜 + 배송 단계만 / 연락처·배송지는 비공개)
```

- 같은 그룹이 서로의 배송 단계를 보며 단톡방 대화거리 생성 ("철수 지금 출발했네?")
- 안 받은 사람의 신청 조급함 유도
- 프라이버시: 장소(배송지)는 그룹에도 노출 안 함

### 그룹 특성

- 비공개(링크 아는 사람만), 인원 제한 없음, 다른 그룹 정보 열람 불가

### ⚠️ 중요 — 날짜 슬롯은 전체 통합 관리

- 그룹 무관 전체 날짜 풀(7/6~10/16, 103일)에서 하루 1건
- 그룹 A가 8/5 예약 시 그룹 B도 8/5 불가
- 조회 시 `group_id` 무관, `date` 컬럼만으로 가용 체크

## 10. 입력값 Validation

- **이름:** 빈 값 방지, 최소 2자
- **연락처:** 010-XXXX-XXXX 형식 검증 + 입력 중 자동 하이픈
- **장소:** 빈 값/최소 길이 체크 (의미없는 입력 방지)
- **날짜:** 마감일 버튼 비활성화, 범위 밖 노출 안 함
- **시간대:** 미선택 시 진행 불가
- **예상 인원:** 1명 이상, 상한선(예: 10명)
- **요청사항/메시지:** 선택, 최대 500자
- **동시성:** `deliveries.date`에 unique 제약(status != '취소' 조건). 동시 신청 실패 시 "방금 다른 분이 먼저 신청하셨어요 😢 다른 날짜를 선택해주세요" → 날짜 단계 복귀 + 목록 재조회

## 11. UI/UX 편의성

- **입력값 보존:** 이전 버튼/새로고침/이탈 시 `sessionStorage` 임시 저장, 재진입 시 이어서
- **전송 중 중복 클릭 방지:** 즉시 버튼 비활성화 + 로딩 표시
- **위치 입력:** 카카오 주소검색 API 연동(또는 안내 문구)
- **에러 메시지 톤:** "필수 항목입니다" → "앗, 이건 꼭 알려주셔야 해요! 🙏"
- **접이식 FAQ:** "꼭 신청해야 하나요? → 아니에요, 안 받으셔도 결혼식엔 꼭 와주세요 😊" 등

## 12. Supabase 테이블

### groups

```
id / name / slug / created_at
```

### deliveries (직접 배달)

```
id / group_id / name / phone / location / date(전체 통합 unique)
time_slot / party_size / message / status(대기중/확정/완료/취소)
tracking_stage(주문접수/준비중/배송출발/배송완료)  ← 실시간 배송 현황
review_rating / review_text  ← 배송 완료 후 리뷰 (공개)
met_photo_url / met_at  ← (후순위) 당일 연결용 — 전달 시 사진/만난 일시 기록
created_at / updated_at
```

- 본인 확인 조회: (name + phone 끝4)로 매칭, 충돌 시 date로 구분

### messages (마음 배송)

```
id / group_id / name / stamp / message / created_at
```

### rsvp (참석 의사 전하기)

```
id / name / phone / side(신랑측/신부측) / attending(참석/마음으로)
companions(동반 인원) / children(자녀 수) / kids_meal(유아식 필요)
meal(식사함/안함/미정) / created_at / updated_at
└ (name + phone) unique — 중복 방지, 재제출 시 수정
```

### waiting_list

```
id / name / phone / created_at
```

> 날짜당 1건 (7/6 ~ 10/16, 총 103일, 그룹 무관 전체 통합)

## 13. Admin 페이지 (`/admin`)

- 환경변수 기반 비밀번호 + 세션 로그인
- 그룹 생성/관리 (그룹명 입력 → 고유 링크 자동 생성)
- 주문 목록 (대기중/확정/완료/취소 탭) + 그룹별 필터링
- 마음 배송 메시지 목록 (방명록 관리)
- 날짜별 캘린더 뷰 (전체 슬롯 현황)
- "이번 주 일정" 요약 뷰
- 구글 캘린더 내보내기(.ics)
- 확정 시 SMS 발송 (솔라피 API)
- 대기자 명단 관리
- 긴급 일정 취소/연기 처리 (당일 우천 등 — 확정 건 취소 + 하객 안내 발송)

## 14. 우선순위 (개발 순서 제안)

1. 그룹 테이블 + 전체 통합 날짜 슬롯 로직 (핵심)
2. 배달 메인(메뉴 분기) + 그룹 페이지
3. 직접 배달 스텝 폼 + 요약 확인 + 완료 화면
4. 마음 배송 폼 + 방명록 연동
5. 모바일 청첩장 개편(방명록, 접근 키, 팝업/플로팅 제거)
6. 취소/변경 페이지 + 배송 추적
7. Admin (그룹 생성, 주문/메시지 관리)
8. SMS 연동(솔라피), 영상 링크, 대기자, .ics 등 부가 기능
