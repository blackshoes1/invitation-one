# 유저 플로우 상세 (배달 → 청첩장)

배달 링크가 시작점이고, 모바일 청첩장은 "만남 이후의 보상"으로 공개된다.
아래는 각 경로를 **화면 / 하객 행동 / 저장 데이터 / 상태 전이 / 예외**까지 파고든 문서다.
(→ 실제 구현 기준: RPC·쿼리 파라미터·검증 로직 포함)

---

## 0. 진입 경로 3가지

| 대상 | 받는 링크 | 특징 |
|---|---|---|
| 친구·동료 그룹 | `/delivery/group/[slug]` | 배달 링크만 공유. 청첩장 링크 미노출 |
| (그룹 없음) | `/delivery` | 통합 배달 메인 |
| 어르신·친척 | `/?key=접근키` | 청첩장 직행. 배달 단계 생략 |

- 청첩장 메인(`/`)은 `NEXT_PUBLIC_INVITATION_KEY`가 설정돼 있으면 `?key=`가 맞아야 열림. 아니면 `LockedGate`("아직 공개 전이에요").
- 종이 청첩장 QR에 인쇄할 링크는 `/?key=접근키&from=qr` — `from=qr`가 있어야 본인확인/뱃지 UI가 뜬다.

---

## 1. 배달 메인 — 두 갈래 (`/delivery`, `/delivery/group/[slug]`)

**화면**
1. 인트로 애니메이션 1회(🛵 + "배송 출발" 도장). `sessionStorage`로 1회만, 스킵 가능.
2. 폼 헤더: "🛵 성근영·김아영 스토어" / (그룹명) / 남은 자리 수(전체 통합 기준).
3. 그룹 페이지면 상단에 **그룹 신청 현황**(이름·날짜·배송단계만, 연락처·배송지 비공개) — `get_group_status(slug)`.
4. 메뉴 두 갈래: **직접 배달 받기 🛵**(메인) / **마음 배송 💌**(보조, 작게 아래).
5. 먼저 받은 하객 리뷰 흐름(리뷰 N개 미만이면 숨김 또는 "첫 손님을 기다리고 있어요 🛵").

**분기**: 직접 배달 → §2 / 마음 배송 → §3

---

## 2. 직접 배달 받기 — 스텝 폼

**화면 (진행률 도트 ●●○○)**

| 단계 | 입력 | 검증 |
|---|---|---|
| 1 | 이름 + 연락처 | 이름 2자+, `010-XXXX-XXXX`(자동 하이픈) |
| 2 | 배송지 | 빈 값/최소 길이 |
| 3 | 배송 희망일 (7/6~10/16) | 예약된 날짜 비활성, 범위 밖 숨김 |
| 4 | 시간대 (오전/오후/저녁) | 미선택 시 진행 불가 |
| 5 | 예상 참석 인원 | 1~10명 |
| 6 | 요청사항 (선택) | 최대 500자 |
| → | **요약 확인** → "네, 주문할게요 🛵" | |

**저장** — RPC `create_delivery(group_id, name, phone, location, date, time, party, message)`
- 반환: 새 `id`(=주문 접근 토큰). `status='대기중'`, `tracking_stage='주문접수'`로 삽입.
- 달력 가용성: `get_booked_dates()` (취소 제외 전체 통합).

**예외 — 동시성**
- 같은 날짜를 다른 사람이 방금 선점 → `create_delivery`가 `date_taken`(errcode 23505) 예외.
- UI: "방금 다른 분이 먼저 신청하셨어요 😢 다른 날짜를 선택해주세요" → 날짜 단계로 복귀 + 목록 재조회.
- 방어: `deliveries.date`에 조건부 unique 인덱스(`status <> '취소'`) + 함수 내 사전 체크 이중.

**UX**
- 이전/새로고침/이탈 대비 `sessionStorage` 임시 저장 → 재진입 시 이어서.
- 전송 중 버튼 즉시 비활성 + 로딩(중복 클릭 방지).

### 2-1. 완료 화면 (`CompletePage`)
- 주문번호 #001, 주문 카드(스크린샷 유도 📸), 배송 추적(초기 "주문접수").
- "나 청첩장 배송 신청했다 🛵" 단톡방 공유(Web Share / 링크 복사).
- 신청자 전용 영상(`NEXT_PUBLIC_VIDEO_URL`, 없으면 "준비 중").
- 취소/변경 링크 → `/delivery/manage/[id]`.

### 2-2. 배송 추적 4단계 (`TrackingView`)
`주문접수 → 준비중 → 배송출발 → 배송완료` — Admin이 `tracking_stage`를 수동 전환하면 하객 화면에 반영.
단계별 문구: 준비중("청첩장 곱게 챙기는 중 🎁"), 배송출발("🛵 신랑이 출발했어요"), 배송완료("직접 전해드렸어요 🎉").
※ `status`(대기중/확정/완료/취소)와 **별개** — status는 슬롯·SMS·리뷰 게이팅용, tracking_stage는 연출용.

### 2-3. 만남 → QR → 공개
현장에서 종이 청첩장(공통 QR) 전달 → 하객이 스캔(`/?key=…&from=qr`) → 모바일 청첩장 공개 + 본인확인 뱃지(§5).

### 2-4. 리뷰 (배송 완료 후)
- `status='완료'`가 되면 manage 페이지에 별점(1~5) + 한줄(선택, 100자) 폼 노출.
- RPC `submit_review(id, rating, text)` → `'ok' | 'rating' | 'not_ready'`. 완료 건에만 기록.
- 이미 남겼으면 읽기 전용("리뷰 감사합니다 🙏"). 리뷰는 공개 → 청첩장 축하 피드로.

## 3. 마음 배송 — 메시지 폼 (`HeartForm`)

**화면** — 축하 스탬프(🎉💐🥂❤️🙏) 탭 → 이름 → 한마디(선택) → "마음 전하기 💌".

**저장** — `messages` 테이블 insert(anon). 이름/스탬프/메시지/그룹.

**완료** — "따뜻한 마음 잘 받았어요 🥰" + 마음배송 전용 감사 영상(`NEXT_PUBLIC_HEART_VIDEO_URL`) → **[모바일 청첩장 보기]로 바로 공개**.
남긴 메시지는 청첩장 방명록(축하 피드)에 표시. 뱃지 "마음으로 함께한 분 💌"는 신청 시점에 부여.

**전환** — 마음↔직접 배달은 완료/manage 화면에서 부드럽게 전환 가능(거절이 끝이 아님).

---

## 4. 취소 / 변경 (`/delivery/manage/[id]`)

- `get_delivery(id)`로 조회(=id가 접근 토큰, 로그인 불필요). 실시간 추적 + (완료 시)리뷰 폼 포함.
- **변경**: `reschedule_delivery(id, date, time)` → `'ok' | 'taken' | 'range'`. 기존 슬롯 해제 → 새 슬롯 점유.
- **취소**: `cancel_delivery(id)` → `status='취소'`, 슬롯 즉시 해제(대기자 알림 트리거 대상).
- 상태별 화면: 취소됨/변경완료/완료(리뷰) 분기.

---

## 5. 본인 확인 & 뱃지 (QR 진입자 전용, `VerifyBadge`)

- 노출 조건: `?from=qr` (배달 데이터 없는 어르신 그룹엔 숨겨 혼란 방지).
- 입력: 이름 + 전화 끝 4자리 → RPC `verify_guest(name, last4, ymd?)`.
  - **1건 매칭**: 즉시 인증. "홍길동님, 8/15에 만나서 반가웠어요!" + 뱃지 🛵✅ + (완료 시)"N번째 손님" (`guest_no` = 완료 순번).
  - **2건+ 매칭**(동명이인+끝4 우연 일치): "만나신 날짜를 알려주세요" → `ymd` 추가 입력으로 재조회.
  - **0건**: "성함·번호를 다시 확인해주세요".
- 인증되면 `onVerified(name)` → 축하 피드/지도에서 본인 항목 강조("내 축하", 📍 하이라이트).

## 6. 수렴점 — 모바일 청첩장 (`/`)

섹션 순서: **Hero → 인사 → 갤러리 → D-Day → 오시는 길 → 💝 축하해준 사람들 → 마음 전하기(계좌) → 참석 의사 전하기**

### 6-1. 💝 축하해준 사람들 (`Guestbook`)
- 헤더: "지금까지 N명이 함께해주셨어요" (N = 마음배송 메시지 + 직접배달 완료 핀).
- **🗺️ 지도 뷰**(`JourneyMap`): `get_journey_pins()` 완료 핀. 탭 시 "OO님을 만났어요 · 날짜 · 동/구". 정확한 주소 비노출(앞 2토막). 본인 핀 강조.
- **💬 메시지 뷰**(`MessageFeed`): 방명록(💌) + 리뷰(🛵✅ 별점)를 `created_at` 시간순 통합. 뱃지로 구분, 연락처 비노출, 본인 항목 강조.
- 빈 상태: "첫 손님을 기다리고 있어요 🛵".
- 하단: `?from=qr`일 때만 본인 확인(§5).

### 6-2. 마음 전하기 (계좌)
- 기본 접힘. 신랑측/신부측 각 [계좌번호 복사](통째 복사 + 토스트) + [카카오페이](보조).

### 6-3. 참석 의사 전하기 (`Rsvp`)
- 이름 + 구분(신랑측/신부측) + 참석 여부. 참석 시 동반 인원/자녀·유아식/식사 여부.
- `submit_rsvp(...)` — `(name+phone)` 유니크. 재제출 시 "이미 전해주셨어요 😊 수정할까요?" → 기존 수정.
- 배달 신청과 역할 분리: 배달 = 식 전 만남 / 참석 의사 = 당일 참석·식수 집계.

## 7. 마감 & 대기자

- 슬롯 소진 → 마감 화면 "대기자 명단에 올려드릴까요?" → `waiting_list` insert.
- 취소로 빈자리 발생 → Admin에서 `POST /api/admin/waiting/notify`(전체/개별) → 대기자에게 안내 SMS(+`/delivery` 링크). 솔라피 키 없으면 skip.

## 8. Admin 운영 레이어 (`/admin`)

흐름 위에서 신랑신부가 조작하는 병렬 레이어(환경변수 비밀번호 + 세션).

| 기능 | 동작 |
|---|---|
| 그룹 | 생성 → 고유 slug 링크 자동. 멤버 명단 등록(미신청 표시용) |
| 주문 | 대기중/확정/완료/취소 탭 · 그룹 필터 · 이름·연락처 검색 |
| 상태 전이 | 대기중→확정→완료. **확정/취소 시 신청자 SMS** 자동 |
| 추적 단계 | 주문접수/준비중/배송출발/배송완료 칩 클릭 → 하객 화면 실시간 반영(낙관적 업데이트) |
| 리뷰 | 완료 주문 카드에 ⭐+문구 표시 |
| 캘린더 | 전체 슬롯 월별 뷰 · 이번 주 일정 · `.ics` 내보내기 |
| 대기자 | 전체/개별 빈자리 SMS · 삭제 |
| 방명록 | 마음배송 메시지 목록 |

## 9. 상태 전이 요약

```
deliveries.status:  대기중 ──(Admin 확정, SMS)──▶ 확정 ──(완료)──▶ 완료 ──▶ [리뷰 가능]
                       └─────────────(취소, SMS·슬롯해제)─────────────▶ 취소
deliveries.tracking_stage:  주문접수 ▶ 준비중 ▶ 배송출발 ▶ 배송완료   (Admin 수동, 연출용)
messages(마음배송):  insert 즉시 → 청첩장 방명록 노출 + 💌 뱃지
rsvp:  (name+phone) upsert — 재제출 시 수정
```
